## Behavioral Interview (STAR Method)

1. `"Tell me about a time you disagreed with your manager." Generate a STAR-format answer for a [role] candidate. Situation (2 sentences), Task (1 sentence), Action (3-4 sentences emphasizing diplomacy and data), Result (measurable outcome + what you learned).`
2. `"Describe a situation where you had to make a decision with incomplete information." Create a STAR answer showing: analytical thinking, risk assessment, stakeholder communication, and outcome. Tailor for a [senior/junior] level.`
3. `"Tell me about your biggest professional failure." Write a STAR answer that's genuinely vulnerable (not a humble-brag). Show: accountability, learning, and how behavior changed. Avoid: "I worked too hard" type answers.`
4. `"How do you handle conflicting priorities from multiple stakeholders?" STAR answer demonstrating: prioritization framework, communication transparency, negotiation, and delivering on commitments.`
5. `"Describe a project you led from start to finish." STAR answer highlighting: planning, delegation, risk management, stakeholder updates, course corrections, and measurable business impact.`
6. `"Tell me about a time you had to give difficult feedback." STAR answer showing: empathy, directness, specific examples, constructive framing, and the positive outcome of the feedback.`
7. `"How do you handle ambiguity?" STAR answer demonstrating: comfort with uncertainty, asking clarifying questions, creating structure, iterating, and maintaining momentum without perfect information.`
8. `"Tell me about a time you went above and beyond." STAR answer that doesn't sound like overworking â€” instead shows initiative, customer obsession, or creative problem-solving that created outsized impact.`
9. `"Describe a situation where you had to influence without authority." STAR answer showing: understanding stakeholder motivations, building coalitions, using data, finding win-win solutions.`
10. `"Tell me about a time you had to learn something quickly." STAR answer demonstrating: learning strategies, resourcefulness, applying under pressure, and the compounding benefit of continuous learning.`

## Technical Interview â€” Coding

11. `Explain how you would solve this algorithm problem: [problem]. Walk through: understanding the problem, brute force approach, optimization (time/space analysis), edge cases, and clean implementation. Include Big-O analysis.`
12. `You're asked to implement [data structure] from scratch. Explain: conceptual design, key operations with complexities, trade-offs vs alternatives, and when to use this in practice. Code it with tests.`
13. `Mock a 45-minute coding interview for a [level] [language] position. Problem, hints at 15/30 min marks, optimal solution, follow-up questions the interviewer would ask, and evaluation rubric.`
14. `Given this code: [code block]. Find all bugs, performance issues, and style problems. Prioritize: correctness bugs â†’ security issues â†’ performance â†’ readability â†’ style. Explain each fix.`
15. `Explain [CS concept: hash maps / recursion / dynamic programming / etc.] as if teaching it to a bootcamp student. Use: analogy, visual example, code example, common mistakes, and interview-relevant applications.`
16. `Prepare for a live coding interview. Common patterns to know: two pointers, sliding window, binary search, BFS/DFS, dynamic programming, greedy, backtracking, trie, union-find. One sentence on when to use each.`
17. `Create a study plan for coding interviews at [FAANG company] with [X weeks] of preparation. Weekly schedule: data structures, algorithms, problem count (daily), mock interviews, system design, behavioral prep.`
18. `Write test cases for [function/feature]. Cover: happy path (3), edge cases (5), error handling (3), boundary conditions (3), performance edge (1). Use arrange-act-assert pattern.`
19. `Compare these algorithmic approaches to solving [problem]: brute force, optimized, and most optimal. For each: pseudocode, time complexity, space complexity, and when it's the right choice.`
20. `Explain these Python/JavaScript gotchas that come up in interviews: shallow vs deep copy, mutable default arguments, closure variable binding, floating point precision, integer overflow, string immutability.`

## System Design Interview

21. `Design a URL shortener (like bit.ly). Walk through: requirements (functional/non-functional), API design, database schema, encoding algorithm, high-level architecture, scaling strategy, and trade-offs.`
22. `Design a real-time chat system (like WhatsApp). Cover: message delivery guarantees, WebSocket vs long polling, message storage, read receipts, group messaging, presence system, and horizontal scaling.`
23. `Design a rate limiter for an API gateway. Discuss: token bucket vs sliding window vs leaky bucket, Redis-based implementation, distributed system considerations, configuration per client, and graceful degradation.`
24. `Design a news feed system (like Facebook/Twitter). Cover: fan-out on write vs read, ranking algorithm, caching strategy, notification system, content diversity, and handling celebrity accounts.`
25. `Design a file storage service (like Dropbox/Google Drive). Cover: upload/download, chunking, deduplication, sync across devices, conflict resolution, versioning, and sharing/permissions.`
26. `Design a payment processing system. Cover: idempotency, double-spend prevention, reconciliation, PCI compliance, payment gateway integration, refunds, and transaction monitoring.`
27. `Design a recommendation system (like Netflix/Amazon). Cover: collaborative filtering, content-based filtering, hybrid approach, cold start problem, A/B testing, and real-time vs batch processing.`
28. `Design a key-value store (like Redis). Cover: data structures, persistence (RDB/AOF), replication, partitioning, eviction policies, cluster mode, and common use cases.`
29. `Design a search engine (simplified). Cover: web crawling, indexing (inverted index), ranking (TF-IDF, PageRank), query processing, spell correction, and autocomplete.`
30. `Design a video streaming platform (like YouTube). Cover: video upload pipeline, transcoding, CDN delivery, adaptive bitrate streaming, thumbnails, recommendations, and comments/engagement.`
31. `Design a ride-sharing service (like Uber). Cover: matching algorithm, surge pricing, geospatial indexing, real-time tracking, ETA calculation, payment processing, and driver allocation.`
32. `Design a distributed message queue (like Kafka). Cover: producers/consumers, topics/partitions, ordering guarantees, retention policy, consumer groups, exactly-once semantics.`
33. `Design a monitoring/alerting system (like Datadog). Cover: metric collection, time-series database, dashboard queries, alert rules, notification channels, anomaly detection, and data retention.`
34. `Design an e-commerce platform. Cover: product catalog, search, cart, checkout, inventory management, order processing, and recommendation engine. Focus on high availability during sales events.`
35. `Design a collaborative document editor (like Google Docs). Cover: operational transformation vs CRDTs, real-time sync, cursor tracking, revision history, access control, and offline support.`

37. `"Design a product for [user problem]." Framework: define users â†’ JTBD â†’ list solutions â†’ prioritize features (MoSCoW) â†’ define MVP â†’ success metrics â†’ go-to-market.`
38. `"[Product metric] dropped 20% this week. How would you investigate?" Framework: verify data â†’ segment (device, geo, user type) â†’ check for external factors â†’ hypothesis tree â†’ prioritize investigation â†’ resolution.`
39. `"Should [company] enter [new market]?" Framework: market size â†’ competitive landscape â†’ user need â†’ company fit (capabilities, brand, distribution) â†’ risks â†’ recommendation with confidence level.`

## Leadership & Management Interview

41. `"How do you build and manage a high-performing team?" Discuss: hiring philosophy, onboarding, goal-setting, 1:1 structure, feedback culture, growth plans, handling underperformance, and retaining top talent.`
42. `"Tell me about a team conflict you resolved." STAR answer showing: identifying root cause (not just symptoms), facilitating dialogue, finding common ground, creating accountability, and preventing recurrence.`
43. `"How do you make decisions?" Discuss: one-way vs two-way door decisions, data-driven vs intuition, RACI for decision rights, disagreement and commitment, and decision documentation.`
44. `"What's your approach to technical debt?" Answer: how to quantify it, prioritization framework (risk-based), negotiating with stakeholders, allocating sprint capacity (e.g., 20% rule), and measuring progress.`
45. `"How do you manage up?" Discuss: understanding manager's priorities, proactive communication, managing expectations, giving upward feedback, and being a force multiplier for your manager's success.`

## Salary & Negotiation

91. `"What are your salary expectations?" Prepare 3 responses: redirect to understanding full comp, provide a researched range, and counter-question to learn their budget. Include data sources for salary research.`
92. `Write a counter-offer email for a job paying $[offered] when your target is $[target]. Include: gratitude, market data, unique value proposition, specific number, and flexibility on other components.`
93. `Prepare negotiation talking points for: base salary, equity, signing bonus, annual bonus, title, start date, remote work, and professional development budget. Which are high-leverage levers?`
94. `Create a compensation comparison framework. Total comp: base + equity (vest schedule) + bonus (OTE) + benefits (healthcare, 401k match, PTO, perks). Compare 3 offers side by side.`
95. `Write a "competing offer" negotiation script. Be honest, not manipulative. Show: genuine interest in this company, transparency about the other offer, specific ask, and collaborative tone.`
