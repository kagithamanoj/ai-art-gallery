## Blog & Article Writing

### Technical Blog Post
```
Write a technical blog post about [topic].

Structure:
- Hook: Start with a surprising fact or problem statement
- Context: Why this matters (2-3 sentences)
- Main content: Step-by-step with code examples
- Visuals: Describe diagrams/screenshots to include
- Conclusion: Key takeaways + call to action
- Meta: Suggest title, meta description, and 5 SEO keywords

Tone: Conversational but authoritative. Like explaining to a smart colleague.
Word count: ~1500 words
```

### LinkedIn Post (Viral Format)
```
Write a LinkedIn post about [topic/experience].

Format:
- Opening hook (first line must stop the scroll)
- Short paragraphs (1-2 sentences each)
- Include a personal story or lesson learned
- Add 3-5 relevant hashtags
- End with a question to drive engagement
- Use line breaks between paragraphs
- Total: 150-200 words
```

## Email Writing

### Cold Outreach Email
```
Write a cold outreach email to [recipient type] about [purpose].

Requirements:
- Subject line: Under 50 chars, curiosity-driven
- Opening: Personal hook (reference their work/company)
- Value prop: What's in it for THEM (not you)
- Social proof: One line of credibility
- CTA: One specific, low-friction ask
- PS: Optional conversation starter
- Total: Under 150 words
```

### Follow-Up Email
```
Write a follow-up email for [context: job application / sales / networking].

Tone: Professional but warm, not desperate.
Reference: [previous interaction]
Add value: Share [article/insight/resource] relevant to them.
CTA: Suggest specific next step with a date.
Length: 3-5 sentences max.
```

### Professional Reply Templates
```
Write a professional email reply for this scenario: [scenario]

Tone: [Formal / Friendly-professional / Casual]
Key points to convey:
1. [point 1]
2. [point 2]

Constraints:
- Acknowledge their message
- Be concise (under 100 words)
- End with clear next step
```

## Resume & Career

### Resume Bullet Points
```
Rewrite these job responsibilities as achievement-oriented resume bullets:

[paste responsibilities]

Format: [Action Verb] + [What you did] + [Result/Impact with numbers]

Example: "Led migration of 3 microservices to Kubernetes, reducing deployment time by 60% and infrastructure costs by $40K/year"

Use strong verbs: Led, Architected, Optimized, Reduced, Increased, Automated, Delivered
```

### Cover Letter
```
Write a cover letter for [position] at [company].

My background: [brief summary]
Why this role: [genuine interest]
Key achievements to highlight:
1. [achievement 1]
2. [achievement 2]
3. [achievement 3]

Tone: Confident, specific, enthusiastic (not generic)
Length: 250-300 words
Structure: Hook ‚Üí Why them ‚Üí Why me ‚Üí Why now ‚Üí CTA
```

### Interview Answer (STAR Format)
```
Help me answer this interview question using the STAR method:

Question: "[interview question]"

Context:
- Situation: [what was happening]
- Task: [what you needed to do]
- Action: [what you specifically did]
- Result: [outcome with metrics]

Make it concise (60-90 seconds when spoken), specific, and highlight my skills in [relevant skills].
```

## Creative Writing

### Story Opening
```
Write the opening paragraph of a [genre] story with these elements:
- Setting: [time/place]
- Protagonist: [brief description]
- Conflict hint: [what's about to go wrong]
- Mood: [atmosphere]

Make the first sentence impossible to stop reading.
Use sensory details (sight, sound, smell, texture).
Show, don't tell emotions.
```

### Product Description
```
Write a compelling product description for [product].

Target audience: [who]
Key features: [list]
Emotion to evoke: [excitement/trust/urgency]
Tone: [premium/casual/technical]

Structure:
1. Headline (benefit-focused, under 10 words)
2. Subheadline (expands on headline)
3. 3 feature bullets with benefits
4. Social proof line
5. CTA
```

Include:

- Contributing guidelines

### API Documentation
```
Document this API endpoint:

Endpoint: [method] [path]
Purpose: [what it does]

Generate:
- Description
- Parameters table (name, type, required, description)
- Request body schema (with example)
- Response schema (with example for success + error)
- cURL example
- Python requests example
- Error codes table
```
